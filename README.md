# 急診排班系統（CP-SAT Scheduler）

本專案是一套以 **Google OR-Tools CP-SAT** 為核心的急診醫師排班系統，包含：

- 前端網頁：設定月份、人員、預假與每日人力需求
- Python FastAPI 後端：使用 CP-SAT 求解排班
- 自動排班：自動產生白班（O）與夜班（N）
- 手動編輯：可再調整 O / N / R / C / S 等班別
- 即時統計：顯示每日班別數量、個人工時與整體工時差異

---

## 1. 專案檔案

```text
project/
├─ scheduler(by CP-SAT).html   # 前端操作介面
├─ app.py                      # FastAPI + OR-Tools CP-SAT 後端
└─ README.md                   # 本說明文件
```

前端會呼叫：

```text

```

因此使用自動排班前，必須先啟動 `app.py`。

# 5. 自動排班操作流程

## 5.1 選擇排班年月

頁面上方的「年月」選擇欲排班月份。

若未指定，系統會自動帶入 **下個月**。

更換月份後，系統會依當月天數重新建立排班設定。

---

## 5.2 設定員工人數

在「員工人數」輸入：

```text
8 ～ 12 人
```

預設為：

```text
10 人
```

修改後按：

```text
更新員工清單
```

系統會依人數建立人員欄位。

---

## 5.3 修改員工姓名

在「員工管理與請假設定」中，可直接修改每位人員姓名。

例如：

```text
王醫師
李醫師
張醫師
```

姓名修改後會同步顯示於排班表。

---

## 5.4 輸入預假日期

每位員工可輸入預假日期，使用半形逗號分隔。

例如：

```text
5,12,20
```

代表該員工：

- 5 日預假
- 12 日預假
- 20 日預假

系統會將這些日期視為不可排班。

### 注意

只需要輸入「日期數字」，不需要輸入年月。

正確：

```text
3,8,21
```

不建議：

```text
2026/8/3, 2026/8/8
```

---

## 5.5 設定每日人力需求

「每日人力需求設定」可分別設定每天：

- 白班人數
- 夜班人數

預設：

```text
白班：2 人
夜班：2 人
```

每月最後一天的夜班固定為：

```text
0 人
```

因此最後一天的夜班欄位不可修改。

可依實際需求調整，例如：

```text
1 日：白班 3 人 / 夜班 2 人
2 日：白班 2 人 / 夜班 2 人
3 日：白班 2 人 / 夜班 1 人
```

---

## 5.6 設定求解秒數

「求解秒數」代表 CP-SAT 最多可以運算多久。

範圍：

```text
1 ～ 60 秒
```

預設：

```text
15 秒
```

建議：

- 一般排班：10～15 秒
- 條件較複雜：20～30 秒
- 希望嘗試更好的最佳化結果：30～60 秒

求解時間越長，不代表一定會得到不同結果，但 CP-SAT 有更多時間尋找較佳解。

---

## 5.7 執行自動排班

完成上述設定後，按：

```text
🎯 執行自動排班
```

系統會將以下資料送到 Python 後端：

- 人員名單
- 預假日期
- 每日白班需求
- 每日夜班需求
- 當月天數
- 最大求解秒數

運算期間會顯示：

```text
CP-SAT 深度運算中
```

進度條是依求解秒數推估的時間進度，後端目前沒有即時傳回真正的 Solver 搜尋百分比。

排班完成後會顯示成功訊息。

# 6. 自動排班規則

目前 Python CP-SAT 後端實際套用以下限制。

## 6.1 每人每天最多一個班

同一位員工同一天不能同時排：

```text
白班 + 夜班
```

---

## 6.2 預假日不可排班

若某人設定：

```text
12 日預假
```

則 12 日：

```text
白班 = 不可排
夜班 = 不可排
```

---

## 6.3 預假前一天不可排夜班

例如：

```text
12 日預假
```

則：

```text
11 日夜班 = 不可排
```

避免夜班工作跨到預假日。

---

## 6.4 夜班後隔天不可接白班

禁止：

```text
10 日 N
11 日 O
```

也就是：

```text
N → O
```

不可連接。

---

## 6.5 最多連續工作 6 天

任意連續 7 天中：

```text
最多只能工作 6 天
```

至少需有 1 天沒有 O / N 班。

---

## 6.6 白班最多連續 3 天

禁止連續：

```text
O O O O
```

因此白班連續最多：

```text
3 天
```

---

## 6.7 每月最後一天不排夜班

最後一天：

```text
N = 0
```

此規則同時存在於：

- 前端每日需求設定
- Python CP-SAT 約束

---

# 7. 排班最佳化目標

在所有硬性規則都成立的前提下，CP-SAT 會盡量最佳化以下項目。

## 7.1 平衡白班數量

盡量縮小：

```text
白班最多者 - 白班最少者
```

權重：

```text
100
```

---

## 7.2 平衡夜班數量

盡量縮小：

```text
夜班最多者 - 夜班最少者
```

權重：

```text
100
```

---

## 7.3 讓相同班別盡量連續

系統會對「新的一段白班或夜班開始」加入懲罰，以減少班別被過度切碎。

例如相較於：

```text
O - O - O
```

系統會傾向：

```text
O O O
```

在其他條件允許時，使排班較集中、較連續。

此項權重：

```text
10
```

# 8. 排班結果閱讀方式

自動排班主要使用：

| 符號 | 說明 |
|---|---|
| O | 白班 |
| N | 夜班 |
| 假 | 預假 |
| 空白 | 未排班 |

顏色：

- O：藍色
- N：綠色
- 假：紅色

# 9. 手動編輯模式

自動排班完成後，可切換：

```text
✏️ 手動編輯
```

若要將自動排班結果帶入手動編輯，按：

```text
🔄 轉換自動排班結果
```

轉換後：

- 自動白班 → O
- 自動夜班 → N
- 預假 → 假
- 未排班 → 空白

---

## 9.1 手動班別

手動模式支援：

| 班別 | 說明 | 工時 |
|---|---|---:|
| O | 白班 | 12 小時 |
| N | 夜班 | 12 小時 |
| R | 急救室 | 12 小時 |
| C | 看診班 | 8 小時 |
| S | 外傷小夜 | 8 小時 |
| 假 | 預假 | 0 |

可直接使用每個日期格子的下拉選單修改。

---

## 9.2 清空所有班別

按：

```text
🗑️ 清空所有班別
```

會清除手動班別，但保留原本設定的預假。

執行前系統會要求確認。

---

# 10. 每日班別統計

手動模式會即時計算每天：

- O 人數
- N 人數
- R 人數
- C 人數
- S 人數

每次修改班別後，統計會重新更新。

---

# 11. 個人工時統計

系統會統計每位員工：

- 白班 O 次數
- 夜班 N 次數
- 急救室 R 次數
- 看診班 C 次數
- 外傷小夜 S 次數
- 總班次
- 總工時
- 平均日工時

工時計算：

```text
O = 12 小時
N = 12 小時
R = 12 小時
C = 8 小時
S = 8 小時
```

若某人的總工時明顯高於或低於平均值，表格會用不同底色提示。

---

# 12. 下方排班統計

排班表下方會顯示：

- 總員工數
- 總班次
- 總工時
- 平均工時
- 最高與最低工時差異

可用來快速觀察整體工作量是否平均。

# 13. 常見錯誤排除

## 問題 1：顯示「無法連線到後端」

錯誤訊息：

```text
❌ 無法連線到後端！請確認 Python app.py 是否已啟動
```

請確認已執行：

```bash
python app.py
```

並確認終端機顯示 Uvicorn 已啟動。

再測試：

```text
http://127.0.0.1:8000/health
```

應回傳：

```json
{"status":"ok"}
```

---

## 問題 2：`ModuleNotFoundError`

例如：

```text
ModuleNotFoundError: No module named 'fastapi'
```

請安裝套件：

```bash
pip install fastapi uvicorn ortools pydantic
```

若使用虛擬環境，請確認虛擬環境已啟用。

---

## 問題 3：排班失敗 / 無解

系統可能顯示：

```text
條件太嚴苛產生邏輯衝突，無法排出！
```

常見原因：

1. 某一天要求的人數大於可上班人數
2. 太多人同一天預假
3. 預假前一天禁止夜班造成夜班人力不足
4. 連續工作上限與白班連續上限造成衝突
5. 人員總數太少，但每日人力需求太高

可嘗試：

- 降低特定日期白班或夜班需求
- 增加員工人數
- 分散預假日期
- 增加求解秒數

注意：

若模型本身沒有可行解，增加求解秒數通常無法解決，必須調整排班條件。

---

## 問題 4：修改月份後資料重設

目前更換月份會重新初始化部分排班資料與每日需求。

因此建議：

```text
先選定月份 → 再輸入預假與需求 → 最後執行排班
```

避免完成設定後才切換月份。

---

## 問題 5：重新整理網頁後排班消失

目前資料只存在瀏覽器記憶體中，尚未寫入：

- 資料庫
- LocalStorage
- JSON 檔案
- Excel

因此重新整理或關閉頁面後，資料可能消失。

正式使用前，建議後續加入「儲存 / 載入排班」功能。

# 16. 目前版本限制

目前系統自動求解器只負責：

```text
O：白班
N：夜班
```

以下班別只存在於手動編輯模式：

```text
R：急救室
C：看診班
S：外傷小夜
```

也就是說，CP-SAT 目前不會自動安排 R / C / S。

另外目前尚未包含：

- 排班資料永久儲存
- Excel / CSV 匯出
- 上月最後幾天班別銜接
- 個別醫師班別偏好
- 指定某人固定班
- 假日與平日不同權重
- 國定假日辨識
- 夜班後固定休息天數
- 每人最低 / 最高總班數
- R / C / S 自動最佳化
- 登入與多人共用
